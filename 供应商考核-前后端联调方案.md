# 供应商考核前后端联调方案（模板前端固定）

## 1. 方案目标

- 前端固定 Excel 样式模板（表头、左侧结构、合并单元格）。
- 后端仅提供动态数据（季度评分、评分依据、评审结论）。
- 通过 `itemDefId` 对齐前端模板中的动态行。

---

## 2. 查询接口（页面初始化/查询）

### 2.1 接口定义

- 方法/路径：`GET /vendorAssess/yearly?vendorCode=V001&year=2026`
- 说明：按供应商+年份返回全年四个季度考核数据

### 2.3 字段约定

- `score`：建议支持 `null`（未评分），避免与真实 `0分` 混淆。
- `displayOrder`：用于前端排序稳定。
- `version`：用于保存时并发控制（乐观锁）。

## 3. 保存评分接口（推荐批量）

### 3.1 接口定义

- 方法/路径：`POST /vendorAssess/score/batch-save`
- 说明：按季度批量保存评分及依据

### 3.2 请求示例

```json
{
  "vendorCode": "V001",
  "year": 2026,
  "quarter": "Q1",
  "version": 12,
  "scores": [
    {
      "itemDefId": 1001,
      "score": 8,
      "scoreBasis": "1批仪表UI缺少"
    },
    {
      "itemDefId": 1002,
      "score": 0,
      "scoreBasis": ""
    }
  ]
}
```

### 3.3 响应示例

```json
{
  "code": 200,
  "msg": "保存成功",
  "data": {
    "success": true,
    "version": 13,
    "updatedAt": "2026-03-05 17:35:00"
  }
}
```

### 3.4 校验规则

1. `quarter` 必须是 `Q1/Q2/Q3/Q4`。
2. `0 <= score <= itemMaxScore`。
3. `scoreBasis` 可空，不做必填校验。
4. `itemDefId` 必须属于该供应商考核项定义。
5. `version` 不一致时返回并发冲突。

---

## 4. 保存评审结论接口

### 4.1 接口定义

- 方法/路径：`POST /vendorAssess/conclusion/save`
- 说明：保存全年评审结论

### 4.2 请求示例

```json
{
  "vendorCode": "V001",
  "year": 2026,
  "version": 13,
  "conclusionText": "A:通过"
}
```

### 4.3 响应示例

```json
{
  "code": 200,
  "msg": "保存成功",
  "data": {
    "success": true,
    "version": 14,
    "updatedAt": "2026-03-05 17:40:00"
  }
}
```

---

## 5. 错误码建议

- `50060`：参数校验错误（分值越界、季度非法等）
- `50061`：业务数据不存在（vendor/year/itemDefId 无效）
- `50062`：并发冲突（version 过期）
- `500`：系统异常

---

## 6. 前端接入约定

1. 前端模板固定，动态单元格通过 `itemDefId` 映射填充。  
2. 每季度动态列映射：
   - 各项分数：`score`
   - 季度得分：`quarterTotalScore`（或按模板位置使用 `deptScore`）
   - 评分依据：`scoreBasis`
3. 查询条件使用：`vendorCode + year`。  
4. 保存策略：
   - 编辑评分：调 `score/batch-save`
   - 编辑结论：调 `conclusion/save`
5. 保存后以前端返回的 `version` 覆盖本地版本号。
